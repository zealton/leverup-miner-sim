Miner Economics / Boundary Simulator

Every parameter below is public. Pull the sliders and find the boundary yourself: mining is profitable when the fleet is below equilibrium N*, and unprofitable above it. The market — not the team — keeps the system pinned to that line, because refueling is a weekly decision every miner makes with their own money.

这是什么机制？机制说明 // MECHANISM

这是 LeverUp「贡献引擎」矿机模型的公开边界模拟器。核心思想：矿机的收益全部来自协议的真实收入回购（零增发），而"是否续费加油"这个每周决策，让市场自己把矿机总量钉在盈亏平衡线 N* 上——无需团队干预。下面所有参数都是公开的，拖动滑块即可自己验证。

01 / 买矿机
机身 + 满油箱
一次性买入（如 $200）：机身永久拥有，燃料按周消耗，烧完需续费。
02 / 挖收益
真实收入分池
协议每周把真实收入（手续费 + 客户亏损）的一部分（如 30%）回购成奖池，按算力占比分给所有矿机。100% 回购、零增发。
03 / 加速
交易或推荐 → 最高 3×
交易或推荐好友可给矿机加速（上限 3×）；完全不活跃的矿机产出会逐周衰减到 0。
04 / 加油决策
市场自动平衡
只有当周收益 ≥ 燃料成本，理性矿工才会续费。矿机太多 → 收益摊薄 → 有人退出 → 收益回升。均衡点就是 N*。
05 / 反刷量
刷量必亏
自己刷手续费，最多只能拿回所付费用的 30%——结构上必然亏损。推荐是唯一的零成本加速方式（有意为之）。

怎么读下面的图：当前矿机数 N 在 N* 左侧（绿色区）= 挖矿有利润、舰队会扩张；在右侧（红色区）= 满速矿机也覆盖不了燃料费、理性矿工停止续费、系统自动收缩。协议端在任何参数下都不会出现 USD 净流出。
